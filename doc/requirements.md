# Chat - 即时通讯应用需求文档

## 一、项目概述

### 1.1 项目名称
Chat - 即时通讯应用

### 1.2 项目目标
构建一个支持 Web 端和桌面端的即时通讯应用，支持一对一聊天、群组聊天，支持多种消息类型，具备完善的用户系统和聊天记录管理功能。

### 1.3 项目结构
Monorepo（pnpm workspace）

```
chat/
├── packages/
│   ├── client/          # 前端（React + Webpack）
│   ├── server/          # 后端（Node.js + Redis + WebSocket）
│   └── shared/          # 前后端共享类型、常量、工具
├── doc/                 # 项目文档
├── pnpm-workspace.yaml
├── package.json
├── tsconfig.json
└── CLAUDE.md
```

## 二、功能需求

### 2.1 用户系统

#### 2.1.1 注册
- 支持用户名注册
- 用户名唯一性校验
- 密码加密存储
- 注册成功后自动登录

#### 2.1.2 登录与认证
- 用户名 + 密码通过 HTTP 请求登录
- 登录成功后服务端返回 JWT Token 和 Session ID
- Token 存储到本地（localStorage），用于下次自动登录
- 自动登录：启动时检测本地 Token，有效则自动登录并获取最新 Session
- Session 机制：每次登录（含自动登录）获取最新 Session ID
- WebSocket 连接时必须携带 Session ID 进行验证，Session 无效则拒绝连接
- 登出功能：清除本地 Token 和 Session，断开 WebSocket

#### 2.1.3 用户搜索
- 按用户名搜索其他用户
- 搜索结果展示用户基本信息
- 支持从搜索结果发起聊天

### 2.2 聊天功能

#### 2.2.1 一对一聊天
- 与单个用户建立私聊会话
- 实时消息收发（基于 WebSocket）
- 在线状态显示
- 消息已读/未读状态

#### 2.2.2 群组聊天
- 创建群组
- 邀请用户加入群组
- 群消息广播
- 群成员列表查看

#### 2.2.3 消息类型

| 类型 | 说明 |
|------|------|
| 文字 | 纯文本消息 |
| 图片 | 支持图片上传、预览、缩略图生成 |
| 录音 | 支持浏览器录音、音频播放 |
| 代码 | 代码块展示，支持语法高亮 |
| Markdown | Markdown 内容渲染展示 |
| 文件 | 任意文件上传与下载 |

#### 2.2.4 聊天记录
- 聊天记录持久化存储
- 历史消息分页加载（滚动加载）
- 聊天记录关键词搜索
- 搜索结果定位到对应消息

### 2.3 缓存
- 前端本地缓存策略（用户信息、会话列表、最近消息）
- 缓存与服务端数据同步机制
- 离线消息支持（上线后同步未读消息）
- 未读消息计数

## 三、非功能需求

### 3.1 多端支持
- **Web 端**：主流浏览器访问（Chrome、Firefox、Safari、Edge）
- **移动端**：PWA + 响应式适配
  - 响应式布局（断点：移动端 <768px / 平板 768-1024px / 桌面 >1024px）
  - 移动端视图切换模式（会话列表 ↔ 聊天窗口全屏切换）
  - 触摸优化（长按替代右键、底部弹出菜单、44px 最小触摸目标）
  - PWA 支持（Service Worker、离线缓存、安装到主屏幕）
- **桌面端**：Electron 打包
  - macOS（优先支持）
  - Windows
  - Linux

### 3.2 数据层设计
- 当前阶段仅使用 Redis 存储所有数据
- 数据访问层采用 Repository 模式抽象
- 预留 MongoDB 适配器接口
- 后期规划：冷数据从 Redis 迁移到 MongoDB，服务启动时从 MongoDB 加载热数据到 Redis

### 3.3 开发规范
- TypeScript 严格模式
- ESLint + Prettier 代码规范
- 每个功能编写对应的测试用例
- 自动化测试（单元测试 + 集成测试）

### 3.4 架构设计原则

本项目是大型长期迭代项目，必须重视兼容性和扩展性，避免新增需求导致大规模重构：

#### 3.4.1 兼容性
- 新功能不得破坏已有功能，接口变更必须向后兼容
- 共享类型的修改需同步评估对前后端的影响
- 数据结构变更需提供迁移方案，不能让旧数据失效

#### 3.4.2 扩展性
- 分层架构：路由层 → 服务层 → 数据访问层，职责清晰
- 接口/抽象定义契约（如 Repository 接口），便于替换实现
- 组件设计为可复用、可组合粒度（如聊天窗口复用于私聊和群聊）
- 配置集中管理，避免硬编码
- 消息类型、事件类型等使用枚举/常量统一管理，新增类型仅需扩展

#### 3.4.3 避免大规模重构
- 新增需求优先通过扩展（新增文件/模块）而非修改已有代码实现
- 关键模块预留扩展点（消息处理管道、中间件链、事件监听）
- 数据模型预留可选字段，避免频繁改动核心数据结构
- 前端状态管理按功能模块拆分 store，避免单一 store 膨胀
- 后端路由按功能模块拆分文件，保持单文件职责单一

### 3.5 版本管理
- 按版本迭代开发（v0.1.0 ~ v1.6.1）
- 每个版本包含明确的功能范围
- 每个版本完成后进行测试验证

## 四、扩展功能需求（v1.2.0+）

### 4.1 AI 智能体桌面应用（v1.2.0）
- 独立 Electron 桌面应用（`packages/agent-app`）
- 管理 AI Agent 连接 chat 中的机器人
- Agent 通过 bot token 轮询消息 → LLM 回复 → 通过 bot API 发回
- 支持 DeepSeek、MiniMax 等 OpenAI 兼容格式的 LLM
- Agent 列表管理（创建/编辑/删除/启停）
- 实时运行日志显示

### 4.2 消息增强（v1.3.0）

#### 4.2.1 消息撤回
- 发送者可在 2 分钟内撤回消息
- 撤回后双方显示 "xxx 撤回了一条消息"
- 已撤回的消息不可再编辑或回应

#### 4.2.2 消息编辑
- 发送者可在 5 分钟内编辑消息内容
- 仅支持文本和 Markdown 类型消息编辑
- 编辑后消息旁显示 "(已编辑)" 标记

#### 4.2.3 引用回复
- 回复消息时引用原消息内容
- 输入区域上方显示引用预览条（可取消）
- 消息气泡内显示引用快照（包含原发送者和内容摘要）
- 保存引用快照防止原消息被删除后丢失

#### 4.2.4 Emoji 选择器
- 消息输入区提供 Emoji 选择按钮
- 弹出 Emoji 选择器浮层
- 选择 emoji 后插入到输入框光标位置

#### 4.2.5 消息表情回应
- 对消息添加表情回应（如 👍❤️😂🎉😮😢）
- 同一 emoji 的 toggle 操作（点击添加/取消）
- 消息下方显示 reaction pills（emoji + 计数）
- hover 消息时显示快捷 reaction 按钮

### 4.3 用户体验与通知（v1.4.0）

#### 4.3.1 用户头像与个人资料
- 用户可上传头像图片
- 编辑昵称（nickname）和个人简介（bio）
- 所有界面中 Avatar 显示真实头像，无头像则 fallback 首字母

#### 4.3.2 已读回执显示
- 自己发送的消息旁显示状态：✓ 已发送、✓✓ 已读
- 私聊中对方已读时显示蓝色双勾
- 群聊中可查看已读/未读成员列表

#### 4.3.3 输入状态指示
- 对方正在输入时显示 "xxx 正在输入…"
- 群聊显示多人输入状态
- 空闲 3 秒后自动停止输入状态

#### 4.3.4 系统通知
- 新消息到达时触发浏览器桌面通知（Browser Notification API）
- 通知内容包含发送者和消息预览
- 点击通知跳转到对应会话
- 未读总数显示在页面标题中
- Electron 端使用原生通知 + dock 角标

#### 4.3.5 暗色模式
- 支持 light/dark/system 三种模式切换
- CSS 变量 + Ant Design ConfigProvider theme 双重切换
- 偏好持久化到 localStorage

### 4.4 会话管理（v1.5.0）

#### 4.4.1 置顶会话
- 置顶会话排在列表最前
- 右键会话可置顶/取消置顶

#### 4.4.2 免打扰
- 免打扰的会话不触发桌面通知
- 会话列表显示免打扰图标

#### 4.4.3 群消息置顶
- 群聊内可置顶某条消息
- ChatWindow 顶部显示置顶消息条

#### 4.4.4 消息转发
- 将消息转发到其他会话（支持多选）
- 转发消息显示 "转发自 xxx" 标记

#### 4.4.5 会话归档/删除
- 归档会话从列表隐藏，可通过归档入口查看和恢复
- 删除会话从用户视角移除

#### 4.4.6 会话标签
- 为会话设置标签（工作/朋友/家人/其他）
- 支持按标签筛选会话列表

### 4.5 Bot 增强（v1.6.0）

#### 4.5.1 Bot 获取聊天历史
- 新增 Bot API 获取会话历史消息
- Agent 启动时自动加载完整聊天历史到上下文
- 使 bot 了解整个聊天内容而非仅直接收到的消息

#### 4.5.2 更多 LLM 提供商
- 支持 OpenAI、Claude、通义千问
- 支持自定义 OpenAI 兼容端点（如 Ollama 本地模型）

#### 4.5.3 Bot 富文本回复
- LLM 回复含 markdown 特征时自动以 markdown 类型发送
- 聊天界面已有 MarkdownMessage 组件支持渲染

#### 4.5.4 Slash 命令
- Bot 支持 /help、/model、/reset、/system 等命令
- Slash 命令由 Agent 本地处理，不发送给 LLM

### 4.6 移动端适配（v1.7.0 / v1.8.0 / v1.9.0）

#### 4.6.1 响应式布局（v1.7.0）
- 建立响应式基础设施（断点变量、mixin、useIsMobile hook）
- 主布局移动端视图切换（会话列表 ↔ 聊天窗口全屏互斥）
- Header 移动端精简（隐藏文字标签，仅图标，高度 56px）
- 认证页面响应式（登录/注册卡片适配小屏幕）
- 触摸基础优化（touch-action、最小点击区域 44px、安全区域适配）

#### 4.6.2 组件移动端适配（v1.8.0）
- ChatWindow 移动端返回按钮 + 精简工具栏
- MessageBubble 气泡宽度移动端 85%
- MessageContextMenu 移动端底部弹出 Sheet + 长按触发
- EmojiPicker 移动端全屏弹出
- ConversationList 触摸友好间距
- 所有弹窗组件移动端全屏化

#### 4.6.3 PWA 支持（v1.9.0）
- PWA Manifest（应用名称、图标、standalone 模式）
- Service Worker（Workbox：静态资源 CacheFirst、API NetworkFirst）
- 离线消息队列（断网时暂存消息，恢复后自动发送）
- "添加到主屏幕" 安装提示
- 网络状态监听（断线提示 + 自动重连）

### 4.7 服务端机器人运行模式（v1.10.0）
- 创建机器人时可选择「服务端运行」或「客户端运行」
- 服务端模式：服务器直接消费消息队列并调用 LLM 回复，无需 agent-app
- LLM 配置在创建时填写（provider/apiKey/model/systemPrompt/contextLength）
- 支持全部 6 种 LLM provider（deepseek/minimax/openai/claude/qwen/custom）
- API Key AES-256-GCM 加密存储，前端回显脱敏
- 对话历史 Redis 持久化，服务器重启不丢失上下文
- 创建后自动运行，列表可编辑配置、暂停/恢复
- 服务器启动时自动恢复运行中的 Bot

### 4.8 远程 Skill 系统（v1.11.0）

#### 4.8.1 概述
- 服务端 Bot 通过标准化 Skill 协议远程控制用户的 Electron 桌面端
- Bot 的 LLM 使用 function calling 决策调用哪个 Skill
- 指令通过 Socket.IO 下发到 Electron 客户端本地执行
- 执行能力集中在 Electron 端（完整 Node.js 环境 + 系统 API）

#### 4.8.2 Skill 标准协议
- 统一的 Skill 接口定义（name/description/parameters/platform/permissions）
- Skill 参数使用 JSON Schema 描述，与 LLM function calling 天然匹配
- 支持异步执行和超时控制

#### 4.8.3 内置 Skill（Mac 平台）
- 备忘录（Notes.app）：列出/搜索/读取/创建/更新/删除笔记
- 日历（Calendar.app）：查看/创建/删除日程
- 提醒事项（Reminders.app）：增删改查
- 文件操作（Finder）：搜索/打开/移动/复制/压缩
- 相册（Photos.app）：列出相册/搜索/导出照片
- 剪贴板：读写剪贴板内容
- Shell 命令：执行任意终端命令
- 浏览器：打开 URL、获取当前标签页信息
- 系统信息：CPU/内存/磁盘/网络状态
- 系统通知：发送桌面通知

#### 4.8.4 安全机制
- 权限分级：read（自动执行）、write（单次确认）、execute（单次确认）、dangerous（逐次确认 + 命令预览）
- Skill 白名单：用户可配置允许/禁止的 Skill
- 审计日志：所有 Skill 执行记录可查
- Bot 权限绑定：每个 Bot 可配置允许使用的 Skill 子集
